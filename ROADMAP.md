## Phase 108: 소싱처 실시간 모니터링 🔄
- `SourceMonitorEngine`: 소싱처 등록 → 주기적 체크 → 변동 감지 → 자동 대응 → 알림
- `SourceChecker` ABC + Amazon/타오바오/1688/쿠팡/네이버 체커 mock
- `ChangeDetector`: 가격변동/품절/삭제/셀러비활성 감지 + 심각도 자동 분류
- `AutoDeactivationService`: 소싱처 문제 시 내 판매채널 자동 비활성화
- `AlternativeSourceFinder`: 대체 소싱처 자동 검색 + 매칭 점수 (가격40%+평점30%+배송20%+안정성10%)
- `SourceMonitorScheduler`: 인기상품 30분/일반 2시간/비활성 24시간 주기 모니터링
- `SourceMonitorDashboard`: 활성/비활성/문제 현황 + 변동 피드
- API: `/api/v1/source-monitor` (16개 엔드포인트)
- 봇 커맨드: `/source_status`, `/source_check`, `/source_alerts`, `/source_dead`, `/source_alternatives`, `/source_dashboard`
- 관련 코드: `src/source_monitor/`, `src/api/source_monitor_api.py`, `tests/test_source_monitor.py`

## Phase 109: 판매 채널 자동 연동 (상품 등록·수정·삭제 동기화)
> **핵심 가치**: 소싱처 변동 시 판매채널도 자동 반영 — 소싱→판매 완전 자동화
- `ChannelSyncEngine`: 소싱처 상품 변동 → 판매채널 자동 반영 오케스트레이션
- `ChannelPublisher` ABC + 구현체:
  - `CoupangPublisher`: 쿠팡 상품 등록/수정/삭제/상태변경 mock
  - `NaverPublisher`: 네이버 스마트스토어 상품 관리 mock
  - `InternalPublisher`: 자체몰 상품 관리
- `ProductMapper`: 소싱처 상품 → 판매채널 상품 매핑 (제목 번역, 가격 변환, 이미지 변환)
- `ListingStatusManager`: 판매중/일시중지/삭제 상태 관리 (108 AutoDeactivation 연동)
- `SyncConflictResolver`: 동시 수정 충돌 해결 (소싱처 우선 / 판매채널 우선)
- `ChannelSyncDashboard`: 채널별 동기화 현황, 실패 건, 대기 건
- API: `/api/v1/channel-sync`
- 봇 커맨드: `/sync_status`, `/sync_channel`, `/sync_force`, `/listing_status`

## Phase 110: 실시간 마진 계산기 강화 (적자 판매 방지)
> **핵심 가치**: 원가+배송+관세+수수료+환율 전부 포함 실시간 마진 계산, 적자 상품 자동 경고
- `RealTimeMarginCalculator`: 실시간 마진 계산 (원가+해외배송+관세+부가세+국내배송+플랫폼수수료+결제수수료+환율손실)
- `PlatformFeeCalculator`: 플랫폼별 수수료 자동 계산 (쿠팡/네이버/자체몰 수수료 구조)
- `MarginAlertService`: 마진율 임계값 알림 (적자/저마진/목표미달)
  - 적자(마진<0%) → 즉시 알림 + 자동 판매 중지 제안
  - 저마진(마진<5%) → 경고 알림
  - 목표미달(마진<설정값) → 정보 알림
- `MarginSimulator`: 가격/환율/배송비 변경 시 마진 영향 시뮬레이션
- `ProfitabilityRanking`: 상품별 수익성 순위 (상위/하위 N개)
- `MarginTrendAnalyzer`: 마진율 추이 분석 (일별/주별/월별)
- API: `/api/v1/margin`
- 봇 커맨드: `/margin <sku>`, `/margin_alert`, `/profit_ranking`, `/margin_simulate`

## Phase 111: 경쟁사 가격 모니터링 (자동 가격 조정 제안)
> **핵심 가치**: 동일 상품 다른 셀러 가격 추적, 가격 경쟁력 유지
- `CompetitorTracker`: 동일/유사 상품 판매하는 경쟁 셀러 가격 추적
- `CompetitorMatcher`: 상품 매칭 (제목 유사도 + 이미지 해시 + 가격대 + 카테고리)
- `PricePositionAnalyzer`: 내 가격 포지션 분석 (최저가/평균/최고가 대비)
- `PriceAdjustmentSuggester`: 가격 조정 제안 (최저가 매칭/마진 유지/중간값)
  - 자동 모드: 규칙 기반 자동 가격 조정 (Phase 33/97 연동)
  - 수동 모드: 제안만 하고 승인 후 적용
- `CompetitorDashboard`: 경쟁 현황, 가격 분포, 내 순위
- API: `/api/v1/competitor-pricing`
- 봇 커맨드: `/competitors <sku>`, `/price_position`, `/price_suggest`, `/competitor_dashboard`

## Phase 112: 고객 주문 ↔ 소싱 자동 매칭 (이행 가능성 사전 확인)
> **핵심 가치**: 주문 접수 시 소싱처 자동 매칭 + 이행 불가 사전 차단
- `OrderSourceMatcher`: 고객 주문 → 소싱처 자동 매칭 엔진
- `FulfillmentChecker`: 이행 가능성 즉시 확인 (재고/가격/배송 가능 여부)
  - 이행 가능 → 자동 구매 프로세스 시작 (Phase 101 연동)
  - 이행 불가 → 고객 즉시 알림 + 대안 제시 또는 자동 취소
- `SourcePriorityManager`: 소싱처 우선순위 관리 (주 소싱처 + 백업 소싱처)
- `OrderRiskAssessor`: 주문 리스크 평가 (소싱처 안정성, 가격 변동성, 배송 리스크)
- `FulfillmentSLATracker`: 이행 SLA 추적 (주문→구매→입고→발송 각 단계 기한)
- API: `/api/v1/order-matching`
- 봇 커맨드: `/match_status <order_id>`, `/fulfillment_risk`, `/sla_status`

### 📦 재고/소싱 안정성 그룹 (113~115)

## Phase 113: 재고 가상화 시스템 (복수 소싱처 통합 재고)
> **핵심 가치**: 소싱처 재고를 가상 재고로 관리, 품절 리스크 최소화
- `VirtualInventoryManager`: 복수 소싱처 재고를 하나의 가상 재고로 통합
- `SourceInventoryAggregator`: 소싱처별 재고 수집 + 합산/가중 계산
- `StockFailoverService`: 주 소싱처 품절 시 자동으로 백업 소싱처 전환
- `VirtualStockCalculator`: 가상 재고 = Σ(소싱처별 실재고 × 신뢰도 가중치)
- `InventoryReservation`: 주문 시 가상 재고 예약 → 구매 확정 시 차감
- `StockForecast`: 소싱처 재고 소진 예측 (판매 속도 기반)
- API: `/api/v1/virtual-inventory`
- 봇 커맨드: `/vstock <sku>`, `/vstock_sources <sku>`, `/stock_failover`

## Phase 114: 셀러 성과 리포트 (내 스토어 종합 분석)
> **핵심 가치**: 월간 자동 리포트 — 의사결정 지원
- `SellerPerformanceEngine`: 내 스토어 종합 성과 분석
- `SalesReport`: 매출/순이익/마진율/주문수/반품률/고객만족도 종합
- `ProductProfitability`: 상품별 수익성 랭킹 + 적자 상품 식별
- `ChannelPerformance`: 판매 채널별 성과 비교 (쿠팡 vs 네이버 vs 자체몰)
- `SourcePerformance`: 소싱처별 성과 (안정성, 평균 마진, 문제 발생률)
- `AutoReporter`: 일간/주간/월간 자동 리포트 생성 (텔레그램 발송)
- `ActionableInsights`: 데이터 기반 개선 제안 ("이 상품 마진 3% 미만, 가격 조정 필요")
- API: `/api/v1/seller-report`
- 봇 커맨드: `/my_report`, `/daily_summary`, `/product_rank`, `/channel_compare`

## Phase 115: 소싱 자동 발굴 (트렌드 기반 신상품 추천)
> **핵심 가치**: 팔릴 만한 상품을 자동으로 찾아서 제안
- `TrendDiscovery`: 마켓플레이스 인기 상품/급상승 트렌드 자동 수집
- `OpportunityScorer`: 소싱 기회 점수화 (수요×마진×경쟁도×소싱난이도)
- `NicheAnalyzer`: 틈새 시장 분석 (수요 높음 + 경쟁 적음 + 소싱 가능)
- `AutoSourcingSuggester`: 일일 소싱 추천 리스트 자동 생성 (상위 N개)
- `TrendAlertService`: 급상승 트렌드 실시간 알림
- API: `/api/v1/sourcing-discovery`
- 봇 커맨드: `/trend_today`, `/sourcing_suggest`, `/niche_find`, `/opportunity_score <keyword>`

### 🔒 안정성/규정 그룹 (116~118)

## Phase 116: 규정 준수 자동화 (통관/세금/라벨링)
> **핵심 가치**: 수입 규정 자동 확인, 통관 리스크 사전 차단
- `ComplianceChecker`: 상품별 수입 규정 자동 확인 (KC 인증, 식약처, 관세청)

- `ImportRestrictionChecker`: 수입 금지/제한 품목 자동 필터링
- `ComplianceReport`: 규정 준수 현황 리포트
- API: `/api/v1/compliance`

## Phase 117: 멀티 계정 관리 (마켓플레이스 계정 통합)
> **핵심 가치**: 여러 마켓플레이스 계정을 하나의 시스템에서 통합 관리
- `AccountManager`: 마켓플레이스별 계정 CRUD + 인증정보 암호화 관리
- `AccountHealthMonitor`: 계정 건강도 모니터링 (평점, 지연률, 취소율, 경고)

- `CrossAccountAnalytics`: 계정 간 성과 비교 + 통합 리포트
- API: `/api/v1/accounts`
- 봇 커맨드: `/accounts`, `/account_health`, `/account_switch`

## Phase 118: 재해 복구 + 고가용성 (HA)
> **핵심 가치**: 시스템 장애 시에도 주문/구매 프로세스 중단 없음
- `DisasterRecoveryManager`: 장애 감지 → 자동 전환 → 복구
- `DataReplicator`: 핵심 데이터 실시간 복제 (주문/재고/소싱처)
- `GracefulDegradation`: 부분 장애 시 핵심 기능만 유지 (주문접수, 알림)
- `RecoveryPlaybook`: 장애 유형별 자동 복구 시나리오
- `HAHealthCheck`: 각 컴포넌트 건강 상태 실시간 체크
- API: `/api/v1/ha`
- 봇 커맨드: `/system_health`, `/failover_status`, `/recovery_test`

### 📈 성장/확장 그룹 (119~120)

## Phase 120: SaaS 플랫폼화 (다른 셀러에게 시스템 제공)
> **핵심 가치**: 이 시스템 자체를 SaaS로 제공 — 수익 모델 확장
- `SaaSOnboarding`: 신규 셀러 가입 → 자동 환경 설정 → 튜토리얼
- `PlanManager`: 구독 플랜 관리 (Free/Starter/Pro/Enterprise)
  - Free: 상품 50개, 주문 100건/월
  - Starter: 상품 500개, 주문 1,000건/월

- `UsageBilling`: 사용량 기반 과금 (API 호출, 소싱처 체크, 주문 처리)

- `SaaSAnalytics`: 테넌트별 사용 현황 + 이탈 예측 + 업셀 제안
- `AdminSuperPanel`: 전체 테넌트 관리 슈퍼 관리자 패널
- API: `/api/v1/saas`
- 봇 커맨드: `/tenants`, `/tenant_stats`, `/billing_status`
